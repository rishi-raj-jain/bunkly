  - Old code used locator.isVisible({ timeout }) which in Playwright 1.59 is an

  wasn't visible at the exact instant of the call, it returned false
  immediately.

  [data-testid="booking-card-BC-PAST01"] card (seeded, always exists after fresh
   seed, no seeded review), uses waitFor({ state: 'visible' }) which properly
  retries. No loop needed.

  REV-04 (tests/reviews.spec.ts):
  - Added test.describe.configure({ mode: 'serial' }) — without this, REV-03
  runs in parallel and can delete a Paris review from the DB while REV-04's
  toggleReviewVote action is mid-flight on the same review, causing a null
  dereference crash ([0].c! on undefined).
  - Changed the vote assertion from toHaveText(initialText) (Playwright uses
  substring matching for strings, so it would falsely match when helpfulCount=0)
   to toHaveClass(/text-accent/) which checks the voted state directly via CSS
  class.

  Server action (src/lib/actions/reviews.ts):
  - Old code inlined [0].c! inside .set() with no null guard — throws if the
  review was deleted between the vote-check query and the update.
  - New code fetches the review count upfront with a null check, returns { 
  error: "Review not found." } gracefully instead of crashing the server action
  and triggering Next.js's error boundary.
